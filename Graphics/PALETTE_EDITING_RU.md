# Руководство по редактированию палитровых PNG

После обновления `WayForward_LYR-Extract.py`, файл `Full.png` теперь сохраняется в **палитровом режиме** (8-bit Indexed Color) с чанком **PLTE**, сохраняя оригинальную палитру из игры.

## Почему это важно?

### Раньше (RGBA):
```
_metatile.png (8-bit, палитра) → Full.png (32-bit RGBA) → редактирование → упаковка
                                          ↓
                                   Палитра потеряна! ❌
                                   При упаковке создается новая палитра
                                   Количество уникальных тайлов меняется
                                   Текстуры в игре битые
```

### Теперь (Paletted):
```
_metatile.png (8-bit, палитра) → Full.png (8-bit, та же палитра) → редактирование → упаковка
                                          ↓
                                   Палитра сохранена! ✅
                                   При упаковке используется оригинальная палитра
                                   Количество тайлов остается правильным
                                   Текстуры в игре корректные
```

## Как правильно редактировать Full.png

### ✅ GIMP (РЕКОМЕНДУЕТСЯ)

#### Открытие:
1. `File → Open` → выберите `Full.png`
2. Проверьте: `Image → Mode` должен быть **Indexed** ✅
3. Посмотрите палитру: `Windows → Dockable Dialogs → Colormap`

#### Редактирование:
- Используйте инструменты рисования (Pencil, Paintbrush)
- **Выбирайте только цвета из палитры!** (справа в Colormap)
- НЕ используйте градиенты, размытие, сглаживание

#### Если нужно добавить новые цвета:
1. `Image → Mode → RGB` (временно конвертируем)
2. Редактируем/добавляем цвета
3. `Image → Mode → Indexed...`
   - **Generate optimum palette**
   - **Maximum colors: 256**
   - **Dithering: None** ⚠️ ОБЯЗАТЕЛЬНО!
   - **Enable dithering of transparency: НЕТ**

#### Сохранение:
1. `File → Export As...`
2. Формат: **PNG**
3. Настройки экспорта:
   - **Interlacing: NO**
   - **Save background color: NO**
   - **Save gamma: NO**
   - **Save layer offset: NO**
   - **Compression level: 9** (по желанию)

---

### ✅ Aseprite (ИДЕАЛЬНО ДЛЯ ПИКСЕЛЬ-АРТА)

#### Открытие:
1. `File → Open` → выберите `Full.png`
2. Aseprite автоматически определит Indexed режим
3. Посмотрите палитру справа

#### Редактирование:
- Рисуйте пиксель-арт инструментами Aseprite
- Используйте только цвета из палитры
- Pixel Perfect mode: включен
- Dithering: **выключен**

#### Сохранение:
1. `File → Save` или `File → Export`
2. Формат: **PNG**
3. Настройки:
   - **Color Mode: Indexed** (оставить как есть)
   - **Palette: Keep current palette**
   - **Resize: NO**

---

### ✅ Photoshop

#### Открытие:
1. `File → Open` → выберите `Full.png`
2. `Image → Mode` должен быть **Indexed Color** ✅

#### Редактирование (осторожно!):
Photoshop плохо работает с Indexed режимом для редактирования. Лучше:

1. `Image → Mode → RGB Color` (временно)
2. Редактируйте изображение
3. **ОБЯЗАТЕЛЬНО** перед сохранением:
   - `Image → Mode → Indexed Color...`
   - **Palette: Local (Adaptive)**
   - **Colors: 256**
   - **Forced: None**
   - **Transparency: НЕТ галочки**
   - **Matte: None**
   - **Dither: None** ⚠️ КРИТИЧНО!
   - **Amount: 0%**

#### Сохранение:
1. `File → Save As...` или `Export → Export As...`
2. Формат: **PNG**
3. Настройки:
   - **Interlaced: NO**
   - **Compression: максимальная** (по желанию)

---

### ❌ Paint.NET

Paint.NET имеет ограниченную поддержку Indexed режима. Используйте только если:

1. При открытии файл остается 8-bit
2. При сохранении:
   - `File → Save As` → PNG
   - **Bit Depth: 8-bit** ✅
   - **Dithering Level: 0** (без дизеринга)

---

### ❌ MS Paint / Paint 3D

**НЕ ИСПОЛЬЗУЙТЕ!** Эти редакторы автоматически конвертируют в 24-bit RGB и добавляют сглаживание.

## Проверка после редактирования

### Python скрипт для проверки:
```python
from PIL import Image

img = Image.open('STATUS/Full.png')
print(f"Режим: {img.mode}")  # Должно быть 'P'
print(f"Размер: {img.size}")
if img.mode == 'P':
    palette = img.getpalette()
    if palette:
        print(f"Палитра: {len(palette)//3} цветов ✅")
    else:
        print("Палитра отсутствует ❌")
else:
    print(f"ОШИБКА: Режим {img.mode}, ожидался 'P' ❌")
```

### Ожидаемый результат:
```
Режим: P
Размер: (512, 256)  # или ваш размер
Палитра: 256 цветов ✅
```

### Если ошибка:
```
Режим: RGBA  # или RGB
❌ Палитра потеряна! Пересохраните правильно.
```

## Типичные ошибки

### ❌ Ошибка 1: Конвертация в RGB
**Симптом**: После упаковки количество уникальных тайлов изменилось (274 → 263)

**Причина**: Редактор конвертировал PNG в RGB/RGBA, палитра потеряна

**Решение**: Пересохраните в Indexed режиме с оригинальной палитрой

### ❌ Ошибка 2: Дизеринг включен
**Симптом**: Текстуры "шумные", много мелких точек разных цветов

**Причина**: При конвертации в Indexed был включен dithering

**Решение**: Повторите конвертацию с Dithering: None

### ❌ Ошибка 3: Сглаживание (Anti-aliasing)
**Симптом**: Края объектов размыты, появились промежуточные цвета

**Причина**: Использовались инструменты со сглаживанием

**Решение**: 
- GIMP: используйте Pencil вместо Paintbrush
- Aseprite: включите Pixel Perfect
- Отключите anti-aliasing во всех инструментах

### ❌ Ошибка 4: Добавлены новые цвета без ограничения
**Симптом**: Палитра > 256 цветов или цвета не из оригинальной палитры

**Причина**: Рисовали произвольными RGB цветами

**Решение**: 
- Используйте только цвета из палитры справа (Colormap)
- Или конвертируйте в Indexed с ограничением 256 цветов

## Workflow для безопасного редактирования

### Рекомендуемый процесс:

1. **Распаковка**:
   ```bash
   python WayForward_TS-Extract.py STATUS
   python WayForward_LYR-Extract.py STATUS
   ```
   Результат: `STATUS_metatile.png` (8-bit, PLTE) и `STATUS/Full.png` (8-bit, PLTE)

2. **Резервная копия**:
   ```bash
   cp STATUS/Full.png STATUS/Full_BACKUP.png
   ```

3. **Редактирование** в GIMP/Aseprite:
   - Открыть `Full.png`
   - Проверить режим: Indexed ✅
   - Редактировать только цветами из палитры
   - Сохранить с Dithering: None

4. **Проверка**:
   ```python
   python -c "from PIL import Image; img=Image.open('STATUS/Full.png'); print(f'Mode: {img.mode}, Palette: {\"YES\" if img.mode==\"P\" and img.getpalette() else \"NO\"}')"
   ```
   Ожидается: `Mode: P, Palette: YES`

5. **Генерация метатайлов**:
   ```bash
   python WayForward_MetatileGenerator.py --map STATUS/Full.png
   ```

6. **Упаковка**:
   ```bash
   python WayForward_TS-Pack.py STATUS_metatile_updated.png --output STATUS --metatile-count 101
   ```

7. **Сравнение**:
   Сравните количество уникальных тайлов до и после. Они должны быть близки!

## Дополнительные советы

### Экспорт палитры для использования в других редакторах

**GIMP**:
```
Windows → Dockable Dialogs → Colormap
Right-click → Export Palette... → сохранить как .gpl
```

**Aseprite**:
```
Palette → Save Palette → сохранить как .aseprite-palette
```

### Если нужно работать в RGB, а затем вернуться

1. Сохраните `.gpl` палитру из оригинала (GIMP)
2. Работайте в RGB режиме
3. Перед упаковкой:
   ```
   Image → Mode → Indexed
   Palette → Custom → загрузите .gpl файл
   Dithering: None
   ```

## Часто задаваемые вопросы

**Q: Можно ли добавить новые цвета в палитру?**  
A: Да, но помните что максимум 256 цветов. Добавляйте новые цвета, затем конвертируйте в Indexed с 256 цветами без дизеринга.

**Q: Почему после редактирования текстуры битые?**  
A: Скорее всего палитра была потеряна при редактировании. Проверьте что `Full.png` в режиме 'P' (Indexed).

**Q: Можно ли использовать онлайн-редакторы?**  
A: Крайне не рекомендуется. Они обычно конвертируют в RGB и добавляют артефакты.

**Q: Что делать если редактор не поддерживает Indexed режим?**  
A: Используйте GIMP или Aseprite. Они специально созданы для работы с палитровой графикой.

---

**Вывод**: Используйте GIMP или Aseprite, сохраняйте в Indexed режиме без дизеринга, и все будет работать идеально! 🎨✅
